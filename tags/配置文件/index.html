    02-如何在.net core 根据环境（开发、测试）自动加载对应的配置文件

    2018-01-16
    .netcore, 后端, 微服务, 配置文件

    一、问题描述

    开发过程中我们经常会遇到这样的问题：开发环境一个配置文件，测试环境一个配置文件，线上环境一个配置文件，一旦有小伙伴不小心签入了配置文件；就会导致各种数据、接口异常。当我们采用了前后端分离后，这个问题会更加明显，前端项目也需要去配置各种环境下的接口地址。

    二、想法

    有了上面的问题，我们就有了一些想法：是不是可以让程序自动根据环境信息来读取对应的配置文件呢，这样不就解决了环境和配置文件的一一对应么？开发小伙伴再也不担心签入了错误的配置文件被喷了；当前这里的前提是：开发者只能调整开发环境配置文件（appsettings.Development.json），线上环境配置文件（appsettings.Production.json）只能由测试配置。

    三、netcore微服务 如何实现这一想法

    1. Startup.cs文件调整
    2. 新增配置文件

    3. 如何配置
    • appsettings.json 默认配置文件
    • appsettings.Production.json 生产环境配置文件
    • appsettings.Development.json 测试环境配置文件
    4. 如何实现

    经过上面3步配置后，需要特别注意

    1. 开发环境使用：dotnet run
    2. 生成环境使用：dotnet publish
      即可加载对应的配置文件
